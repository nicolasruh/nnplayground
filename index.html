What is this?

In this app, you can experiment with simple neural networks and visually inspect every aspect of what is happening under the hood. Several sample projects covering classical machine learning tasks are included. You can also create your own custom projects and save or load them using a plain-text JSON structure.

Limits

Since this app runs entirely in your browser, the maximum size of the network architecture is limited (e.g. no more than 11 output neurons). Even within these limits, you may notice that training or testing projects with many neurons or large numbers of samples can become quite slow. For substantially larger projects, you would need a more powerful environment in which all aspects of the network are defined programmatically.

Learning

There are many different paradigms in machine learning, including reinforcement learning, self-organizing learning, autoencoders, and others. This app focuses exclusively on the most fundamental paradigm: supervised learning with backpropagation. "Supervised learning" means that each training sample is associated with a target, i.e. the desired output for a given input. The learning process consists of three basic steps. First, the difference between the target and the actual output of the network (the loss) is calculated. Second, this loss is propagated backwards through the network, allowing the contribution of each individual weight to the overall error to be determined, as well as the direction in which the weight should be adjusted to reduce the loss (using gradient descent). Third, the weights are updated accordingly. The learning rate determines how aggressively these updates are applied. The weights may be updated after each individual training sample or after an entire batch of samples has been processed (e.g. after one epoch). This process is repeated many times until, ideally, only a negligible discrepancy remains between the target values and the actual outputs of the network.

Stable learning

In theory, even standard neural networks can learn any mapping between inputs and outputs because it can be proven that there exists a configuration of weights capable of implementing the desired function. However, this does not imply that the training process will actually find such a configuration. Stable and efficient learning depends on many additional factors, including the network architecture, the initialization of the weights, the choice of activation functions, the learning rate, and the quality of the training data.

Deep learning

This app supports architectures with up to three hidden layers. However, since all layers are fully connected, this does not really constitute "deep learning." Instead, it demonstrates that simply stacking multiple fully connected hidden layers often makes learning more difficult rather than more effective. Modern deep learning architectures typically include specialized layers, such as convolutional layers, that perform dedicated preprocessing tasks like feature extraction, edge detection, or spatial filtering. Such architectures are beyond the scope of this app, which is limited to fully connected "vanilla" neural networks.

Error function

The error is defined as \(C = \frac{1}{2 \times sampleCnt}\sum^{sampleCnt}_{m=1}(\sum^{outputSize}_{n=1}(neruon_n-target_n)^2)\). In words: Error is defined as \((value - target)^2\). To get error of neural network for one training sample, you simply add errors of all output neurons. The average cost is then defined as average error of all training samples.

Forward propagation

Let's say that the value of connection is the connection's weight (how wide it is) times the first connected neuron. To calculate the value of some neuron you add the values of all incoming connections and apply the sigmoid function to that sum. Other activation functions are possible, but I have not implemented them yet.

Back propagation

The error function defined above is a function dependend on weights of connections in the same way as \(f(x, y) = x^2 + y^2\) is dependend on x and y. In the beginning, the weights are random. Let's say x = 5 and y = 3. The error at this point would be 25 + 9 = 34, which we want to get to 0. Now we take the derivate with respect to each of these weights, which tells us how to adjust the wieghts to minimize the function. \(\frac{\partial f(x, y)}{\partial x} = 2x\), \(\frac{\partial f(x, y)}{\partial y} = 2y\). Now that we have the derivatives, we know the "direction" in which to change the weights. \(x_{new} = x_{old} - rate \times 2x = 5 - 0.1 \times 2 \times 5 = 4\) and that's a little bit closer to the desired 0 result of f(x, y). The rate is necessary to avoid stepping over the minimum.

In practice is the computation of the derivatives is a little bit harder, but all you need to know is the chain rule. I highly recommend 3blue1brown's series and this paper for better understanding.

Choose one of the preconfigured example projekts:

  • basic: simple network with 3 inputs, 1 hidden layer with 3 neurons and 2 outputs
  • perceptron: basically just one (output) neuron with two inputs
  • XOR: simple 2x2x1 network that can learn the XOR task
  • 2x2: simple 4x3x1 network, includes all 16 possible (descrete) input samples
  • 3x3: simple 9x4x1 network, includes 4 of the 2^9 possible (descrete) input samples
  • 4x4: simple 16x4x1 network, includes 4 of the 2^16 possible (descrete) input samples
  • 5x5: simple 25x25x25 network, includes 4 of the 2^25 possible (descrete) input samples
  • smileys: 1600x11x1 network for the task of deciding if a smiley drawn on a 40x40 pixel grayscale canvas is happy or not. 11 examples included, 5 of which are earmarked for testing only
  • MNIST-mini: 256x12x10 network for the task of recognizing handwritten digits from the MNIST dataset (16x16 pixels each). 1000 examples included, 20% of which are earmarked for testing only
All examples can be explored and modified. You can also save your own projects to a pure text format (JSON) and load them later (by pasting the JSON).
